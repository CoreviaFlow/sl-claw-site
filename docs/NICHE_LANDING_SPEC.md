# NICHE_LANDING_SPEC — інструкція як зробити продаючий лендинг

Кожна ніша в каталозі = **самостійний sales-лендинг** (`/n/<slug>/` для RU,
`/ua/<slug>/` для UK). Шаблон один (`seo-build.js → page()`), значить
**виправлення тут множиться × 250**. Цей документ — checklist того, що
**мусить бути на кожній сторінці**, щоб вона продавала, а не була просто
SEO-візиткою.

Базується на QA-прогоні 2026-05-27 (qa-conversion) — підтверджені CRO-блокери
з попередньої версії шаблону, які виправлено в коміті
`feat(niche): CRO patch — trust strip + guarantee + closer FAQ + ...`.

---

## 0 · Принцип

Людина в інтернеті:
- За **3 секунди** має зрозуміти **що це** і **в чому користь для неї**.
- За **15 секунд** має бачити **скільки коштує**, **як купити**, **чи можна повернути**.
- За **30 секунд** мусить мати **точку довіри** (хтось вже купив, або хоч би «125+ ніш у каталозі»).

Якщо хоч одне з цих трьох не закрито на сторінці — ми втрачаємо її назавжди.

## 1 · Структура сторінки (обов'язкові секції)

| # | Секція | Де (зліва направо, зверху вниз) | Призначення |
|---|---|---|---|

| **6** | **Closer block** | "Доводить угоду до грошей" | 4 buller-point — на відміну від звичайного автовідповідача |
| **7** | **Caps** | "Що ще вміє AI-продавець" | 9 каналів/SPIN/RAG/мови — універсальні фічі |
| **8** | **FAQ** | "Часті запитання" | **closer-FAQ (3) → enrichFaq → faqItems** — closer завжди першим |

| **11** | **dev-deploy** (side) | `<details>` закритий за дефолтом | docker compose інструкція — лише для технарів, не лякати власника бару |

| **14** | **Knows chips** (side) | soft-tags | експертиза ніші |

Якщо потрібно — лише в `<details class="dev-deploy">` згорнутому.

## 2 · Hero — що має бути на першому екрані

```html
<div class="np-head">
  <span class="tag {archetype}">{archetype-label}</span>
  <h1>AI-продавець для «{niche}» — продає в листуванні 24/7</h1>
  <p class="lead">{tagline — 1 речення з niches.json}</p>
  <ul class="trust-strip">
    <li>✓ Разово · без абонплати</li>
    <li>✓ ШІ за фактом · ~$5 ≈ 2000 діалогів</li>
    <li>✓ Готовий за годину</li>
    <li>✓ Гарантія повернення 14 днів</li>
  </ul>
  <p class="mono">діалог → кваліфікація → зняття заперечень → угода · 24/7</p>
</div>
```

**Правила:**
- H1 завжди починається з «AI-продавець для» — це консистентно по всіх 125 нішах
  + у нас SEO-якір для «AI-продавець + ніша».
- Tagline — 1 коротке речення, до 80 символів. Беремо з `niches.json: {niche}.tagline`.
- **Trust-strip — обов'язково above the fold.** Це блокер №1 у попередньому шаблоні
  (qa-conversion: «0 trust signals для продукту $249-$999»). Без нього навіть
  ідеальний контент нижче не врятує.
- Pipe-mono показує що це **продавець по етапах**, а не bot-приймальник.

## 3 · box-price — найважливіша side-секція (CRO heart)

Кожна правка тут множиться × конверсія. Що **мусить** бути:

```
[маленький label] цена / акційна цена
[великий] $249  (якщо знижка: $499 закреслено + −50%)
[мала нота] Бот купуєш разово. ШІ платиш за фактом через токен SL-CLAW:
            ~$5 ≈ 2000 діалогів. Без абонплати.
[бейдж] 🛡 Гарантія повернення 14 днів     ← КРИТИЧНО, в emerald-кольорі
[row] Галузь:    {sector}
[row] Можлива ціль: {cta-text}
[CTA primary] Отримати бота                ← веде на /checkout.html?niche=X&tier=Y
[CTA secondary, underline] Запитати у консультанта   ← відкриває Дарину
[compare-hint] Також доступні: Core+Study $449 · Pro $499 ★ (−50%)
               · [Порівняти тарифи →](/pricing.html)
[trust-list] 4 буллети (разово / по факту / повернення / година)
```

**Правила:**
- **Гарантія повернення 14 днів** — обов'язковий бейдж у emerald-стилі, **не сховати в trust-list дрібним**. Це знімає головне заперечення «а раптом не підійде».
- **Pay-per-use нота** ясно проговорена: бот разово + ШІ за фактом + НЕМАЄ абонплати. Унікальна USP яка раніше була зашита тільки в pricing.html.
- **Compare-hint** з ★ на Pro — decoy effect: показує що Pro це премієм-вибір. Без цього anchor-tariff не працює.
- CTA primary прямо веде на checkout з предзаповненим tier — не змушуй людину перейти на /pricing.

## 4 · FAQ — closer-FAQ завжди першим

Топ-3 заперечення які закривають угоду (закодовано в `seo-build.js → t.closerFaq`):

1. **«А якщо не підійде — можна повернути?»** → Гарантія 14 днів, повернення за 24 год.
2. **«А абонплата не виросте?»** → Абонплати нема. Бот разово + ШІ за фактом.
3. **«А якщо я не подужаю налаштувати?»** → Self-service за годину АБО «Під ключ» від $2000.

Ці 3 ВЖЕ закодовано в шаблоні (`UI.{lang}.closerFaq`), додаються автоматично
**ПЕРШИМИ** в `<div class="faq">` блок перед `enrichFaq` і `faqItems`.

Виділені візуально: `.faq-item.faq-closer { border-left: 3px solid #c7d2fe }` —
покупець бачить це не як «загальні питання», а як «питання-блокери, ось відповіді».

## 5 · Соціальний доказ — поки немає реальних відгуків, не вигадуй

**Чого НЕ робити:**
- Стокові фото клієнтів. Брехливі testimonials. Логотипи компаній з якими не працюємо.

**Що можна:**
- Соц-сигнали через числа: «125+ ніш у каталозі», «5 архетипів продажів», «3 рівні підготовки бота».
- Цифри ринку ніші (з `niches.json: depth.market[]`) — Statista, World Bank, Eurostat.
- E-E-A-T блоки: «Хто стоїть за продуктом» — `LEGAL.entity` (ТОВ Корвія Флоу + ЄДРПОУ + email).
- Демо в `phoneHTML()` — живий приклад діалогу замість абстрактних обіцянок.

**Коли з'являться реальні кейси/відгуки:**
- Додати секцію `<section class="np-testimonials">` між FAQ і Blog.
- Шаблон в `seo-build.js → renderTestimonials(n)` (поки не реалізовано).
- Структура у `niches.json`: `testimonials: [{name, role, niche, text, result}]`.

## 6 · Мобільний — `<600px`

- **Не показувати docker-compose блок** — у `<details>` за замовч. згорнутий ✓
- **Trust-strip компактний** (`@media (max-width:600px)` зменшує padding+font) ✓
- **box-price після main** (mobile-grid) — права колонка йде вниз
- **Mobile sticky bar** — `mobile-bar.js` додає нижню панель з «Консультація + дія» (<600px)
- **CTA primary** в side-box має `width:100%` — кнопка через всю ширину mobile

**Чого НЕ робити:**
- Hero з `docker compose up -d` як приклад «розгортання» — лякає нетехнічних
- Дрібний текст <14px без `@media font-size:16px+` — iOS зумить
- Tap-targets <44px (кнопки, посилання в навбарі)

## 7 · SEO + AI-краулери

Уже зроблено в шаблоні:
- `<title>` з ніша-локалізацією + brand suffix
- `<meta description>` + `<meta keywords>` нішеві
- `<link rel="canonical">` + `<link rel="alternate" hreflang>` для всіх гео
- JSON-LD: **Product + BreadcrumbList + FAQPage** (4-х schema'а) — Rich Results готові
- `cover.svg` — Google Images-friendly
- llms.txt у корні, sitemap.xml + sitemap-blog.xml
- Робить blog-дрип через `publish.js` (~50 постів/нішу, hreflang ru↔uk парами)

**Не зламати:**
- Не видаляти JSON-LD з FAQ — Rich Results залежать від `closerFaq + enrichFaq + faqItems`
- Lastmod-cache через `lastmod-cache.json` — sitemap оновлюється лише при реальній зміні контенту
- Якщо додаєш нову секцію — додай її до hash-input для `lastmodFor()`

## 9 · CRO-чеклист (run на кожну зміну шаблону)

- [ ] Trust-strip є above the fold? (видно без скролу на 1440×900)
- [ ] Гарантія 14 днів видно на сторінці хоч раз без скролу до футера?
- [ ] CTA primary («Отримати бота») доступний за 1 клік на десктопі і на mobile?
- [ ] Pay-per-use «$5 ≈ 2000 діалогів» згаданий ясно (не сховано)?
- [ ] Compare-hint показує всі 3 тарифи + Pro має ★?
- [ ] Closer-FAQ (3 запитання) є **першими** в FAQ блоці?
- [ ] docker-compose блок прихований у `<details>`?
- [ ] Mobile trust-strip компактний (`<600px` font 12-13px, gap 6-8px)?
- [ ] CTA-кнопки ≥44×44px на mobile?
- [ ] Жодних фейкових testimonials / stock photos?
- [ ] Дата акції -50% має дедлайн (`PROMO_ON` дивиться у дату) — не «вічна» fake urgency?
- [ ] JSON-LD `FAQPage` включає всі 3 closer-FAQ?

Наступний раунд CRO покращень (поки не реалізовано):
